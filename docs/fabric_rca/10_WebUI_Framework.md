# Fabric Controller
# Engineering Design Specification (EDS)

# 10. Web UI Framework

**Version:** 1.0

**Status:** Living Engineering Design Specification

---

# 1. Purpose

The Web UI Framework provides the visualization layer of the Fabric Controller platform.

Its responsibility is to transform engineering results into an intuitive investigation experience for engineers, architects, and management.

Unlike traditional dashboards that display raw telemetry, the Fabric Controller UI presents progressively refined engineering information:

- Executive health
- Engineering verdict
- Investigation evidence
- Detailed technical analysis

The UI is intentionally separated from engineering computation.

It never performs RCA.

It only visualizes engineering results generated by upstream frameworks.

---

# 2. Responsibilities

The Web UI Framework is responsible for:

- Presenting engineering results
- Rendering RCA reports
- Visualizing congestion analysis
- Displaying engineering reasoning
- Supporting entity navigation
- Providing executive summaries
- Displaying engineering evidence

The UI is NOT responsible for:

- Stress execution
- Telemetry collection
- Evidence correlation
- Engineering reasoning
- Root cause analysis

- Architects

- Overall Verdict

- Software Engineers

- RCA Investigators

Focus

- Interfaces
- Queues

# 13. Visualization Principles

The dashboard follows several design principles.

## Progressive Disclosure

High-level summaries appear first.

Detailed engineering data follows.

---

## Single Source of Truth

The dashboard renders only:

```
rca_ui_report.json
```

No calculations are performed inside the UI.

---

## Traceability

Every engineering conclusion should be traceable back to supporting evidence.

---

## Separation of Concerns

Business logic remains outside the UI.

The UI only presents information.

---

## Consistency

Each dashboard section follows a consistent visual style.

# 18. Design Principles

## Principle 1

The UI presents engineering information.

It never performs engineering analysis.

---

## Principle 2

Executive and engineering views serve different audiences.

---

## Principle 3

Every displayed conclusion should originate from Engineering Reasoning.

---

## Principle 4

Visualization should remain independent of backend implementation details.

---

## Principle 5

New dashboard sections should consume existing normalized artifacts whenever possible.

---

# 19. Summary

The Web UI Framework is the presentation layer of the Fabric Controller platform.

It transforms structured engineering results into a clear, explainable investigation experience for both executive and engineering audiences while maintaining strict separation from analysis and reasoning logic.
